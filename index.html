HTML

  • The head element contains the metadata information about the webpage.
  • The body element represents the visible content shown to the user and holds all the contect.
  • The title element sets the title of your page, which is the title that appears in the browser tab the page is loaded in.
  • The DOCTYPE tells the browser what document type should be opened.
  • The meta element defines metadata such as the character set, description, keywords, author, and viewport.
  • The image tag has 2 attributes -src, the images path and -alt attribute, the descriptive text.
  • The br element creates a break or space between content.
  • The href attribute points to the url for a link.
  • The footer element appears at the bottom of the page and usually contains the author, copyright, contact, sitemap, and navigation.

CSS

  • The selector is the HTML element name at the start of the ruleset. It defines the element(s) to be styled
  • Attribute selector are element(s) on the page with the specified attribute.
  • Class sector are element(s) on the page with the specified class. Multiple instances of the same class can appear on a page.
  • The declartion is a single rule like color: red;. It specifies which of the element's properties you want to style.
  • The selector points to the HTML element you want to style.
  • The declaration block contains one or more declarations separated by semicolons. Each declaration includes a CSS property name and a value, separated by a colon. Multiple CSS declarations are separated with semicolons, and declaration blocks are surrounded by curly braces

GIT

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add *insert document* adds a file as it looks now to your next commit (stage)
  • git commit -m *descriptive message* commits your staged content as a new commit snapshot
  • git clone *insert url* retrieves an entire repository from a hosted location via URL
  • git branch -r displays a list of the remote branches in Git repo.
  • pwd command writes to standard output the full path name of your current directory
  • ls command is used to list all files

JAVASCRIPT

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Strings are data types that can store a series of characters or numbers
  • numbers and booleans do not have quotations around them
  • Array is a single variable that contains a list of values. The index of the first item in the array is 0.
  • A for loop will repeat a block of code and each time increment a counter until the counter no longer satisfies a given condition.
  • Loops allow us to control how many times a set of intructions should be repeated
  • A function is declared by providing a name, a list of inout parameters, and a block of code enclosed in curly brackets. A functions statement begins with the word function followed by the function name.
  • Javascript errors can be viewed in a browsers developer tools, inside the console tab.
  • The console--developers can output messages to the console log to make sure their code is executing as expected.
  • The console tab contains two areas--Console output log: Displays system and user messages from the javascript execution. The console command line interface: Accepts any javascript instructions and executes them immediately
